# Investment-potential-tracking

### 1. Usage

Download the project folder to your local machine, navigate to the project root directory, and add `release_ipt_latest/code` to your MATLAB path. Then call the provided entry scripts (for example, `run_ipt_fixed_test.m`) from that directory to obtain results. The final outputs are stored in the following two variables:

- `cum_wealth`: T × 1 Double — cumulative wealth series
- `b_history`: N × T Double — portfolio weight matrix

### 3. IPT configuration for `release_ipt_latest/results`

The `.mat` files under `release_ipt_latest/results` are produced by the script `release_ipt_latest/code/export_olps_mat_results_all.m`. The main settings are:

- Split rule: dev/test split with `dev_ratio = 0.6`. The dev segment is used for warm-up and parameter selection, and the exported results correspond to the tail test segment (last 40% of the sample, “tail40”).
- Global fixed parameters shared by all datasets: `win_size = 5`, `epsilon = 100`, `tran_cost = 0.001`, `L_smoothing_alpha = 0.2`.
- IPT hyperparameters (per dataset): `weight_inspect_wins`, `risk_inspect_wins`, `L_percentile`, `q_value`, `reverse_factor`, `risk_factor`, `Q_clip_max`, and `max_turnover` are taken from the best-per-dataset row of the summary CSV generated by `run_ipt_fixed_test.m` (see `release_ipt_latest/code/results_fixed_params/*.csv`).
- Q clipping: the internal IPT state `Q_factor` is clipped to the range `[-Q_clip_max, Q_clip_max]` (Q-clip). Setting `Q_clip_max = Inf` disables clipping. The released IPT results correspond to the clipped version, with `Q_clip_max` given in the summary CSV for each dataset.
- Output format: each IPT file `ipt-<dataset>_tail40-*.mat` follows the OLPS layout and contains at least `cum_ret`, `cumprod_ret`, `daily_ret`, `ra_ret`, `run_time`, and `daily_portfolio`.
